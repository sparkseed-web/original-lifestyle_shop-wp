# オリジナルサイト(架空の雑貨ショップ)（WordPress テーマ）



## 概要
架空の雑貨ショップのオリジナルサイトを WordPress テーマとして再構築したものです。

静的サイトとして作ったデザインを、  
「管理画面から簡単に編集できるサイト」に育てていくことを目指しました。

---

## 🌸 このテーマを作った目的
- Wordpressの環境構築の手順を学ぶ
- 静的サイトを WordPress 化する流れを学ぶ  
- メインループを使って管理しやすいサイトを作る
- テンプレート階層や PHP の基本を理解する
- お問い合わせメールを送受信できるようにする  
- 静的サイトをそのまま WordPress に落とし込む

---

## 🌱 使用技術
- 言語: HTML / CSS / jQuery / PHP
- CMS: WordPress:6.9.4
- プラグイン: Advanced Custom Fields（ACF）/ Contact Form 7 / Custom Post Type UI
- 開発環境: VS Code / Git /Local (旧 Local by Flywheel)  
           Local 動作環境: Web Server: nginx / PHP: 8.2.27/ Database: MySQL 8.0.35 
                        

必要なところだけ PHP を使いながら、  
WordPress の仕組みに合わせてテーマを組み立てています。

---

## 🌼 このテーマでできること

### ■ 商品を管理するための投稿タイプ
- `product`  
雑貨やインテリアを登録するための投稿タイプです。

### ■ 商品カテゴリの分類
- `product_category`  
カテゴリーごとに商品を整理できます。

### ■ ニュースを管理するための投稿タイプ
- `news`  
新商品の入荷情報や、ショップからのお知らせを登録するための投稿タイプです。

### ■ カスタムフィールドの活用 (ACF)
Advanced Custom Fields（ACF） を使用し、商品ごとに異なる商品仕様(スペック)を管理画面から直観的に入力できるように構築しました。
商品ごとに、次のような仕様を入力できます。

- サイズ  
- 座面高さ  
- 素材  
- カラー  
- 質量  
- 耐荷重  
- 組み立て  
- 生産国  
- 耐熱温度  
- 注意事項  
- 容量  
- ページ数  
- 表紙  
- 冊数  
- 用紙  
- 価格

雑貨や本、食器など、  
さまざまなアイテムに対応できるようにしています。

## こだわったポイント  
① カスタム投稿タイプを利用して商品カテゴリーをWordpressの管理画面から紐づいて表示できるようにし、HTML、CSSを知らない人がサイトを更新する際でもカテゴリー名の変更や商品の追加を出来るようにしました。  

②  商品の種類によって必要な商品仕様の項目が異なるため、条件分岐(if文)をPHPで実装しました。そうすることでサイト上には各商品の必要な情報だけを表示させることができ管理しやすく、見やすいサイトにすることを意識しました。  

```php
<?php if ( get_field('size') ): ?>  
   <dt>サイズ</dt>  
   <dd><?php the_field('size'); ?></dd>  
<?php endif; ?>
```

このように、カスタムフィールドの値の有無を判定してから出力することで、不要なタグが生成されないようにしています。

## 苦労した点  
- Wordpressの基礎理解
- テンプレート階層の活用
- カスタム投稿タイプとタクソノミーの設計  
初歩的なミスではありますが、「投稿タイプ」と「タクソノミー（カテゴリー）」の役割を混同しており、スラッグ名が混乱してしまうトラブルがありました。

【学びと解決策】
改めてそれぞれの意味・役割を調べ直し、今回のサイトでは以下のように定義することができました。  
・「投稿タイプ」：商品データを格納するメインの箱  
・「タクソノミー」:商品を「ステーショナリー・インテリア」などに分類するための属性  

そしてWordpressは与えられたスラッグ名を元に読み込むファイルを決定するため、「投稿タイプ」は``product``、「タクソノミー」は``product_category``と明確に分けて管理することで意図した通りのカテゴリーページを表示させることができました。


## 💡 補足
このリポジトリには WordPress 本体や `wp-config.php` は含めていません。  
テーマフォルダ（`sotto-site`）のみを管理しています。
